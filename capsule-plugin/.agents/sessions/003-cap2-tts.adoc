== Ce qui a été fait

=== TtsManager.kt — Interface + Implémentations
- `TtsEngine` : interface abstraite (synthesize, isAvailable, name)
- `PiperTtsEngine` : wrapper CLI piper (ProcessBuilder, timeout, stderr)
  - Passe le texte en stdin à piper
  - Conversion WAV → MP3 via ffmpeg (fallback: renommage si pas ffmpeg)
  - Gestion d'erreurs : TtsException si piper introuvable ou exit code != 0
- `NoOpTtsEngine` : fallback placeholder (écrit un fichier texte marqué TTS PLACEHOLDER)
- `TtsException` : runtime exception métier

=== CapsuleExtension — Nouvelles propriétés
- `piperExecutablePath` : chemin vers l'exécutable piper (default: "piper")
- `ttsFallbackEnabled` : active/désactive le fallback noop si piper absent (default: true)

=== CapsuleBuildTask — Refactor
- Remplacement du placeholder inline par l'utilisation de TtsEngine
- Résolution automatique de l'engine : piper → si absent → noop si fallback activé
- Compteurs totalSynthesized / totalFailed en fin d'exécution
- Support injection `internal var ttsEngine` pour les tests
