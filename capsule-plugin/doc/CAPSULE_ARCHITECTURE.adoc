'''
== Vision

Transformer automatiquement un deck reveal.js (produit par slider-gradle)
en une capsule video WebM avec narration TTS synchronisee, sans
post-production humaine.

=== Stack

[cols="3,5"]
|===
| Brique | Role
| slider-gradle | Source unique de verite вЂ” produit le deck reveal.js + script capsule
| Piper | TTS local, offline вЂ” genere un MP3 par slide a partir des notes speaker
| Playwright | Pilote headless Chromium вЂ” controle reveal.js, capture video/audio
| FFmpeg | NON utilise вЂ” WebM natif Playwright, pas de conversion necessaire
|===

'''

== Etapes du processus

=== Etape 0 вЂ” AsciiDoc Source (existant)

Fichier AsciiDoc structure avec balises `[NOTE.speaker]` :

[,asciidoc]
----
== Titre de la slide

Contenu visible...

[NOTE.speaker]
====
Texte que le formateur prononcerait a cette slide.
Devient le script TTS.
====
----

=== Etape 1 вЂ” Generation reveal.js (slider-gradle, existant)

`./gradlew asciidoctorRevealJs` в†’ deck HTML avec :

* `<section>` par slide
* `<aside class="notes">` avec le texte speaker
* attributs `data-transition`, `data-auto-animate`
* config reveal.js complete

=== Etape 2 вЂ” Extraction script capsule (slider-gradle, existant)

`./gradlew asciidocCapsule` в†’ `*-script.txt` :

[,txt]
----
=== CAPSULE SCRIPT : mon-deck ===

--- SLIDE 1 : Agenda ---
Introduce the plan in 2 minutes...

--- SLIDE 2 : First Topic ---
Emphasise the most common misconception...
----

=== Etape 3 вЂ” Synthese TTS (capsule-gradle, a implementer)

[cols="3,5"]
|===
| Input | `*-script.txt` (un segment par slide)
| Output | `slide-01.mp3`, `slide-02.mp3`...
| Moteur | Piper (local, offline)
| Timing | la duree de chaque MP3 в†’ duree d'affichage de la slide
|===

=== Etape 4 вЂ” Injection Audio (capsule-gradle, a implementer)

Option retenue : modifier le deck HTML (Option A) вЂ” plus propre, plus testable.

Le deck modifie contient un `<audio id="audio-N" src="slide-N.mp3">` par `<section>`.
Un script minimal ajoute au deck declare l'autoplay chaine :

[,javascript]
----
// Ajoute au deck modifie
let currentAudio = null;
let currentSlide = 0;
const audios = document.querySelectorAll('section audio');
Reveal.on('slidechanged', (event) => {
  const idx = event.indexh;
  if (currentAudio) currentAudio.pause();
  currentAudio = audios[idx];
  if (currentAudio) {
    currentAudio.currentTime = 0;
    currentAudio.play();
  }
});
audios[0]?.play(); // demarre la premiere slide
----

=== Etape 5 вЂ” Capture Playwright (capsule-gradle, a implementer)

[cols="3,5"]
|===
| Input | Deck reveal.js modifie (avec `<audio>` injectes)
| Action | Playwright pilote Chromium headless, reveal.js gere l'avancement
| Output | `capsule.webm` (video + audio synchro)
| Format | WebM natif, pas de conversion
|===

Script Playwright (pseudo-code) :

[,javascript]
----
const { chromium } = require('playwright');

const browser = await chromium.launch({ headless: true });
const context = await browser.newContext({
  viewport: { width: 1408, height: 792 },
  recordVideo: { dir: './build/capsule', size: { width: 1408, height: 792 } }
});
const page = await context.newPage();

await page.goto('file://' + deckPath);

// Reveal.js gere l'audio via l'event slidechanged
// Playwright attend la fin de chaque audio + transition
// puis appelle Reveal.next()
for (let i = 0; i < slideCount; i++) {
  await page.waitForFunction(`document.getElementById('audio-${i}')?.ended === true`,
    { timeout: 120000 }
  );
  // attendre la transition CSS
  await page.waitForTimeout(500);
  if (i < slideCount - 1) {
    await page.evaluate('Reveal.next()');
  }
}

await page.waitForTimeout(2000); // marge a la fin
await context.close();
await browser.close();
----

=== Etape 6 вЂ” Livrable final

Un fichier `capsule.webm` par deck, lisible par VLC et tous les navigateurs modernes. Aucune post-production requise.

== Contrat slider в†’ capsule

Le contrat entre les deux plugins est minimal :

[cols="3,5"]
|===
| slider expose | capsule consomme
| `build/docs/asciidocRevealJs/*-deck.html` | Deck reveal.js pret a etre pilote
| `build/capsule/*-script.txt` | Script TTS, un segment par slide
| `slides/misc/*.adoc` (source) | Acces direct au texte des notes speaker
|===

Capsule ne modifie JAMAIS les fichiers produits par slider.
Slider n'a aucune dependance sur capsule.

== Decisions Techniques

=== Pourquoi WebM (pas FFmpeg/MP4) ?

* Playwright capture nativement en WebM
* VLC, Chrome, Firefox, Edge lisent WebM
* YouTube accepte WebM
* Toute transco WebMв†’MP4 degrade la qualite (generation de perte supplementaire)
* ZERO dependance FFmpeg dans le pipeline

=== Pourquoi Piper (TTS local) ?

* Offline вЂ” pas de cle API, pas de cout par requete, pas de latence reseau
* Open source вЂ” coherent avec la licence Apache 2.0 du projet
* Voix francaises disponibles (fr_FR)
* API externe (ElevenLabs, OpenAI) en fallback optionnel

=== Pourquoi Playwright (pas Hyperframes/impress.js) ?

* Pilotage natif de reveal.js вЂ” pas besoin de changer de moteur de slides
* Capture video + audio integree dans une seule session
* Licence Apache 2.0
* API JavaScript familiere, executable en npx sans dependance npm projet

=== Pourquoi compileOnly slider-gradle ?

* Decouplage fort вЂ” capsule ne recompile pas slider
* Contrat par fichiers вЂ” le deck HTML et le script texte sont l'interface
* slider reste maitre de son output, capsule reste maitre de sa capture
* Possibilite d'utiliser un autre generateur de slides plus tard (pas de couplage au code)
