'''
== Vision

Transformer automatiquement un deck reveal.js (produit par slider-gradle)
en une capsule video WebM avec narration TTS synchronisee, sans
post-production humaine.

=== Stack

[cols="3,5"]
|===
| Brique | Role
| slider-gradle | Source unique de verite — produit le deck reveal.js + script capsule
| Piper | TTS local, offline — genere un MP3 par slide a partir des notes speaker
| Playwright Java | Pilote headless Chromium — controle reveal.js, capture video/audio (API JVM native, zero npm)
| FFmpeg | NON utilise — WebM natif Playwright, pas de conversion necessaire
|===

'''

  capsule -> piper : tts("texte speaker note")
  piper -> capsule : slide-N.mp3

loop chaque slide
  deck -> deck : joue slide-N.mp3
  deck -> deck : attend 'ended'
  capsule -> playwright : Reveal.next()
  deck -> deck : transition + slide suivante

deactivate deck
playwright -> output : sauvegarde capsule.webm
capsule -> user : ✅ capsule.webm produit
@enduml

== Etapes du processus

=== Etape 0 — AsciiDoc Source (existant)

Fichier AsciiDoc structure avec balises `[NOTE.speaker]` :

[,asciidoc]
----
== Titre de la slide

Contenu visible...

[NOTE.speaker]
====
Texte que le formateur prononcerait a cette slide.
Devient le script TTS.
====
----

=== Etape 1 — Generation reveal.js (slider-gradle, existant)

`./gradlew asciidoctorRevealJs` → deck HTML avec :

* `<section>` par slide
* `<aside class="notes">` avec le texte speaker
* attributs `data-transition`, `data-auto-animate`
* config reveal.js complete

=== Etape 2 — Extraction script capsule (slider-gradle, existant)

`./gradlew asciidocCapsule` → `*-script.txt` :

[,txt]
----
=== CAPSULE SCRIPT : mon-deck ===

--- SLIDE 1 : Agenda ---
Introduce the plan in 2 minutes...

--- SLIDE 2 : First Topic ---
Emphasise the most common misconception...
----

=== Etape 3 — Synthese TTS (capsule-gradle, a implementer)

[cols="3,5"]
|===
| Input | `*-script.txt` (un segment par slide)
| Output | `slide-01.mp3`, `slide-02.mp3`...
| Moteur | Piper (local, offline)
| Timing | la duree de chaque MP3 → duree d'affichage de la slide
|===

=== Etape 4 — Injection Audio (capsule-gradle, a implementer)

Option retenue : modifier le deck HTML (Option A) — plus propre, plus testable.

Le deck modifie contient un `<audio id="audio-N" src="slide-N.mp3">` par `<section>`.
Un script minimal ajoute au deck declare l'autoplay chaine :

[,javascript]
----
// Ajoute au deck modifie
let currentAudio = null;
let currentSlide = 0;
const audios = document.querySelectorAll('section audio');
Reveal.on('slidechanged', (event) => {
  const idx = event.indexh;
  if (currentAudio) currentAudio.pause();
  currentAudio = audios[idx];
  if (currentAudio) {
    currentAudio.currentTime = 0;
    currentAudio.play();
  }
});
audios[0]?.play(); // demarre la premiere slide
----

=== Etape 5 — Capture Playwright (capsule-gradle, a implementer)

[cols="3,5"]
|===
| Input | Deck reveal.js modifie (avec `<audio>` injectes)
| Action | Playwright pilote Chromium headless, reveal.js gere l'avancement
| Output | `capsule.webm` (video + audio synchro)
| Format | WebM natif, pas de conversion
|===

Script Playwright Java (pseudo-code) :

[,kotlin]
----
import com.microsoft.playwright.*
import com.microsoft.playwright.options.*
import java.nio.file.Paths

val playwright = Playwright.create()
val browser = playwright.chromium().launch(
    BrowserType.LaunchOptions().setHeadless(true)
)
val context = browser.newContext(
    Browser.NewContextOptions()
        .setViewportSize(1408, 792)
        .setRecordVideoDir(Paths.get("build/capsule"))
        .setRecordVideoSize(1408, 792)
)
val page: Page = context.newPage()
page.navigate("file://$deckPath")

for (i in 0 until slideCount) {
    page.waitForFunction(
        "document.getElementById('audio-$i')?.ended === true",
        null,
        Page.WaitForFunctionOptions().setTimeout(120_000.0)
    )
    page.waitForTimeout(500.0)
    if (i < slideCount - 1) {
        page.evaluate("Reveal.next()")
    }
}

page.waitForTimeout(2000.0)
context.close()
browser.close()
playwright.close()
----

=== Etape 6 — Livrable final

Un fichier `capsule.webm` par deck, lisible par VLC et tous les navigateurs modernes. Aucune post-production requise.

== Contrat slider → capsule

Le contrat entre les deux plugins est minimal :

[cols="3,5"]
|===
| slider expose | capsule consomme
| `build/docs/asciidocRevealJs/*-deck.html` | Deck reveal.js pret a etre pilote
| `build/capsule/*-script.txt` | Script TTS, un segment par slide
| `slides/misc/*.adoc` (source) | Acces direct au texte des notes speaker
|===

Capsule ne modifie JAMAIS les fichiers produits par slider.
Slider n'a aucune dependance sur capsule.

== Decisions Techniques

=== Pourquoi WebM (pas FFmpeg/MP4) ?

* Playwright capture nativement en WebM
* VLC, Chrome, Firefox, Edge lisent WebM
* YouTube accepte WebM
* Toute transco WebM→MP4 degrade la qualite (generation de perte supplementaire)
* ZERO dependance FFmpeg dans le pipeline

=== Pourquoi Piper (TTS local) ?

* Offline — pas de cle API, pas de cout par requete, pas de latence reseau
* Open source — coherent avec la licence Apache 2.0 du projet
* Voix francaises disponibles (fr_FR)
* API externe (ElevenLabs, OpenAI) en fallback optionnel

=== Pourquoi Playwright Java (pas Hyperframes/impress.js ni npx) ?

* Pilotage natif de reveal.js — pas besoin de changer de moteur de slides
* Capture video + audio integree dans une seule session
* Licence Apache 2.0
* API JVM native (`com.microsoft.playwright:playwright`) — zero npm, zero Node.js
* 100% Kotlin, pas de script MJS externe, pas de npx playwright install chromium
* Browser téléchargé automatiquement par la lib au premier lancement

=== Pourquoi compileOnly slider-gradle ?

* Decouplage fort — capsule ne recompile pas slider
* Contrat par fichiers — le deck HTML et le script texte sont l'interface
* slider reste maitre de son output, capsule reste maitre de sa capture
* Possibilite d'utiliser un autre generateur de slides plus tard (pas de couplage au code)
